# 过滤策略

过滤配置位于 `config/filter-rules.json`，执行脚本为 `scripts/filter-items.js`。

## 规则顺序

1. 校验链接是否存在。
2. 校验分类是否属于 `tech`、`finance`、`news`。
3. 排除明确屏蔽的来源。
4. 排除标题或摘要命中屏蔽词的内容。
5. 排除标题命中低价值模式的内容。
6. 排除标题长度过短的内容。
7. 排除超过保留窗口的旧内容。

## 当前低价值信号

- 促销、优惠券、每日折扣。
- 明显赞助内容。
- 与三类主题无关的娱乐八卦。
- 标题信息量过低，无法判断价值。

## 输出

过滤后的结果写入 `data/processed/filtered-items.json`。保留的记录会带有空的 `filterReasons`，方便后续扩展审计输出。
